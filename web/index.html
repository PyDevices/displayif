displayif

Native display-bus and framebuffer modules for MicroPython. Portable code lives in ports/common/; SoC-specific paths under ports/<mp-port>/. pydisplay board configs that raise NotImplementedError need firmware built with the matching module — natives register directly, with no Python re-export layer in this repo.

CircuitPython has its own display stack (dotclockframebuffer, mipidsi, picodvi, …). displayif does not ship CP bindings — use pydisplay cp_* board configs with CP firmware.

Source on GitHub Hardware bring-up cmods

Native modules

Accelerated on esp32, mimxrt, samd, and rp2

BusDisplay

spibus / i2cbus (common) — SPI and I²C panel buses for pydisplay BusDisplay.

FBDisplay

rgbframebuffer, mipidsi, picodvi, rgbmatrix — framebuffer backends. Parallel RGB uses rgbframebuffer.RGBFrameBuffer (no RGBDisplay).

i80bus

8080-style parallel bus on esp32 (S3), rp2 (PIO+DMA), mimxrt (1062 FlexIO), and samd (SAMD51 GPIO).

Build into MicroPython

Clone as a sibling of micropython/

CMake ports (esp32, rp2)

USER_C_MODULES points at this repo. CMake does not scan siblings the way Make does.

cd micropython/ports/esp32
make submodules BOARD=ESP32_GENERIC_S3
make BOARD=ESP32_GENERIC_S3 \
  USER_C_MODULES=../../../displayif

# Plus other usermods (semicolon list):
make BOARD=ESP32_GENERIC_S3 \
  USER_C_MODULES="/abs/path/to/displayif;/abs/path/to/lv_micropython_cmod"

Make ports (mimxrt, samd, …)

USER_C_MODULES is the workspace parent that contains displayif/.

cd micropython/ports/mimxrt
make USER_C_MODULES=../../.. BOARD=TEENSY41

cd micropython/ports/samd
make USER_C_MODULES=../../.. \
  BOARD=ADAFRUIT_METRO_M4_EXPRESS

ESP32 large framebuffers

RGB and DSI framebuffers prefer PSRAM (MALLOC_CAP_SPIRAM). Enable and size CONFIG_SPIRAM in your board sdkconfig before building — see HANDOFF.md. For a full ESP32-P4 + touch walkthrough, use cmods MP_EXAMPLE.md.
